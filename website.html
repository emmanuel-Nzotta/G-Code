Web Application Architecture:

It describes the relationships between the databases, servers and applications within a system. It can be defined as how different components in a web application is structured. The web application architecture is responsible for fitting all the elements together. There are 2 sides of the web application architecture namely Client – side and child – side. Web application architecture keeps track of all the evolution of how the web application are developed, designed and structured. There are different types of web application architecture namely:

Applications:

There are 4 types of applications namely:

Single - Page Architecture (SPA):

Application based on single page. If user picks your application it does not need to be navigated to other web pages.

Advantage:

SPA is easy to scale and can build highly interactive web applications.

Disadvantage:

SPA limit flexibility with hyperlinks and initial render is slow.

Num2

Progressive Web Application Architecture (PWA):

Application is built on Single Page Architecture by capabilities offline for the user’s web application.

Advantage:

Scalability is simple and apps runs smoothly, with cross platform compatibility.

Disadvantage:

Limits support for management link and SEO and updates offline is more complex.

Num3

Server – side – rendered Architecture (SSR):

The front web page are rendered on the backend server as requested by the user. SSR used for blogs websites.

Advantage:

App are good for SEO websites and when the first page is loaded is happens in an instance.

Disadvantage:

It is not recommended for heavy or complex web pages and apps tend to not focus on the user interface.

Num4

Microservice Architecture:

Focuses on solving an issues with service oriented architecture. Mostly used for bloggers with millions of users.

Advantage:

Apps are highly fault tolerant

Disadvantage:

With smaller projects this architecture will be too much to maintain.
